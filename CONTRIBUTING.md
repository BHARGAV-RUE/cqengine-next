# Contributing

Contributions are welcome! This fork aims to maintain backward compatibility while adding modern features.

## Guidelines

Some general guidelines to follow when contributing:

- **Preserve backward compatibility** - All existing APIs must continue to work
- **Follow Keep a Changelog** - Update `CHANGELOG.md` with your changes
- **Run tests before committing**: `mvn clean test` 
  - Paste the output of SUCCESS with number of tests run as part of the PR
- **Include Docker tests** if modifying persistence/index code
- **Update documentation** as needed
- **Submit a pull request** with a clear description of your changes
- **Be respectful and collaborative** in code reviews and discussions
- **Adhere to coding standards** - Follow existing code style and conventions
- **Write clear commit messages** - Describe what and why, not how
- **Test thoroughly** - Ensure your changes are well-tested
- **Engage with the community** - Participate in discussions and help others
- **Report issues** - Use the issue tracker for bugs and feature requests
- **Stay updated** - Keep your fork in sync with the main repository
- **Seek help when needed** - Don't hesitate to ask for guidance or clarification
- **Respect licensing terms** - Ensure compliance with all relevant licenses
- **Document code changes** - Update inline comments and documentation as necessary
- **Use feature branches** - Create separate branches for each feature or bug fix
- **Review pull requests** - Provide constructive feedback on others' contributions
- **Celebrate contributions** - Acknowledge and appreciate the efforts of all contributors
